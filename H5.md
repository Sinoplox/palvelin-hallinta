## X)
Chacon and Straub 2014
What is git
- Git ei ole kuin muut versiohallinta ohjelmat kuin CVS, Subversion tai Perforce
- Git ajattelee dataa snapshotteina
- Isoin osa toimintoja ovat locaaleita
- Helppo käyttää/tehdä muokkauksia jos ei oo nettii tai yhteyttä resursseihin
- Git tekee checksumin jokaisesta muutoksesta
- Checksum tehty SHA-1 menetelmällä
- Git tallentaa kaikki sen tietokantaan ei nimellä, vain hashillä nimestä
- Git melkein jokaisessa tapauksessa lisää dataa tietokantaan, eikä poista sitä
- Voit menettää jos et ole committanu mutta jos olet se on vaikeaa menettää versiosi
- Gitin tiedostot voivat olla kolmessa eri muodossa: Modified, Staged ja commited
- Staging area/"index" on tiedosto gitin hakemistossa
- Git hakemisto on paikka mihin git tallentaa metadataa

Gitin käyttö ja komennot
- Git add (file) lisää tiedoston sellaisenaan seuraavaan committiin
- Git commit committaa sun nykyisen stagen
- Git pull lataa ja yhdistää commitit etäpaikasta esim. github
- Git push lähettää paikalliset commitit etäpaikkaan esim. github
